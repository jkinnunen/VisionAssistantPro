# Документация по Vision Assistant Pro

**Vision Assistant Pro** — это продвинутый мультимодальный AI-ассистент для NVDA. Он использует лучшие в мире AI-движки для интеллектуального чтения экрана, перевода, голосовой диктовки и анализа документов.

_Это дополнение выпущено в честь Международного дня инвалидов._

## 1. Установка и настройка

Перейдите в **Меню NVDA > Параметры > Настройки > Vision Assistant Pro**.

### 1.1 Настройки подключения
- **Провайдер:** Выберите предпочитаемый AI-сервис. Поддерживаются **Google Gemini**, **OpenAI**, **Mistral**, **Groq** и **Custom** (совместимые с OpenAI серверы, такие как Ollama/LM Studio).
- **Важное примечание:** Мы настоятельно рекомендуем использовать **Google Gemini** для наилучшей производительности и точности (особенно для анализа изображений/файлов).
- **API Key:** Обязательно. Можно ввести несколько ключей (через запятую или с новой строки) для автоматической ротации.
- **Fetch Models:** После ввода API-ключа нажмите эту кнопку, чтобы загрузить актуальный список доступных моделей от провайдера.
- **AI Model:** Выберите основную модель для обычного чата и анализа.

### 1.2 Продвинутая маршрутизация моделей
*Доступно для всех провайдеров, включая Gemini, OpenAI, Groq, Mistral и Custom.*

> **⚠️ Предупреждение:** Эти настройки предназначены **только для опытных пользователей**. Если вы не уверены, что делает та или иная модель, оставьте этот пункт **отключенным**. Выбор несовместимой модели для задачи (например, текстовой модели для Vision) вызовет ошибки и остановит работу дополнения.

Отметьте **"Advanced Model Routing (Task-specific)"**, чтобы получить детальный контроль. Это позволит выбирать конкретные модели из выпадающего списка для разных задач:
- **OCR / Vision Model:** Выберите специализированную модель для анализа изображений.
- **Speech-to-Text (STT):** Выберите модель для диктовки.
- **Text-to-Speech (TTS):** Выберите модель для генерации аудио.
- **AI Operator Model:** Выберите модель для задач автономного управления компьютером.
*Примечание: Неподдерживаемые функции (например, TTS для Groq) будут автоматически скрыты.*

### 1.3 Продвинутая настройка конечных точек (Пользовательский провайдер)
*Доступно только при выборе "Custom".*

> **⚠️ Предупреждение:** Этот раздел позволяет вручную настраивать API и предназначен для **продвинутых пользователей**, запускающих локальные серверы или прокси. Неправильные URL или имена моделей нарушат подключение. Если вы точно не знаете, что делают эти конечные точки, оставьте этот пункт **отключенным**.

Отметьте **"Advanced Endpoint Configuration"**, чтобы вручную ввести данные сервера. В отличие от стандартных провайдеров, здесь нужно **вводить** конкретные URL и имена моделей:
- **Models List URL:** URL для получения списка доступных моделей.
- **OCR/STT/TTS Endpoint URL:** Полные URL для конкретных сервисов (например, `http://localhost:11434/v1/audio/speech`).
- **Custom Models:** Вручную введите имя модели (например, `llama3:8b`) для каждой задачи.

### 1.4 Общие настройки
- **OCR Engine:** Выберите между **Chrome (Быстро)** для быстрых результатов или **AI (Расширенный)** для лучшего сохранения структуры.
    - *Примечание:* Если вы выбрали "AI (Расширенный)", но ваш провайдер — OpenAI/Groq, дополнение интеллектуально направит изображение в Vision-модель вашего активного провайдера.
- **TTS Voice:** Выберите предпочтительный стиль голоса. Список динамически обновляется в зависимости от активного провайдера.
- **Creativity (Temperature):** Управляет случайностью ответов AI. Низкие значения лучше подходят для точного перевода/OCR.
- **Proxy URL:** Настройте, если AI-сервисы ограничены в вашем регионе (поддерживаются локальные прокси, например `127.0.0.1` или мостовые URL).

## 2. Командный слой и горячие клавиши

Чтобы избежать конфликтов клавиш, это дополнение использует **Командный слой**.
1. Нажмите **NVDA + Shift + V** (Главная клавиша) для активации слоя (прозвучит сигнал).
2. Отпустите клавиши, затем нажмите одну из следующих одиночных клавиш:

| Клавиша       | Функция                 | Описание                                                                 |
|---------------|--------------------------|-----------------------------------------------------------------------------|
| **Shift + A** | **AI Operator**         | **Автономное управление:** Дайте AI задание выполнить действие на экране.      |
| **E**         | **UI Explorer**          | **Интерактивный щелчок:** Определяет элементы интерфейса в любом приложении и щёлкает по ним.        |
| **T**         | Smart Translator         | Переводит текст под курсором навигатора или выделенный текст.                        |
| **Shift + T** | Clipboard Translator     | Переводит содержимое буфера обмена.                              |
| **R**         | Text Refiner             | Суммирует, исправляет грамматику, объясняет или выполняет **Пользовательские промты**.                 |
| **V**         | Object Vision            | Описывает текущий объект навигатора.                                     |
| **O**         | Full Screen Vision       | Анализирует весь макет и содержимое экрана.                              |
| **Shift + V** | Online Video Analysis    | Анализирует видео с **YouTube**, **Instagram**, **TikTok** или **Twitter (X)**.  |
| **D**         | Document Reader          | Расширенное чтение PDF и изображений с выбором диапазона страниц.               |
| **F**         | **Smart File Action**    | Контекстно-зависимое распознавание выбранных файлов изображений, PDF или TIFF.          |
| **A**         | Audio Transcription      | Транскрибирует файлы MP3, WAV или OGG в текст.                                |
| **C**         | CAPTCHA Solver           | Захватывает и решает CAPTCHA (поддерживаются гос. порталы).                        |
| **S**         | Smart Dictation          | Преобразует речь в текст. Нажмите для начала записи, снова нажмите для остановки/ввода текста.      |
| **L**         | Status Reporting         | Сообщает текущий прогресс (например, "Сканирование...", "Ожидание").                   |
| **U**         | Update Check             | Вручную проверяет GitHub на наличие последней версии дополнения.                 |
| **Space**     | Recall Last Result       | Показывает последний ответ AI в диалоговом окне для просмотра или уточнения.        |
| **H**         | Commands Help            | Отображает список всех доступных горячих клавиш.                                 |

### 2.1 Горячие клавиши Document Reader (внутри окна просмотра)
- **Ctrl + PageDown:** Перейти на следующую страницу.
- **Ctrl + PageUp:** Перейти на предыдущую страницу.
- **Alt + A:** Открыть диалог чата, чтобы задать вопросы о документе.
- **Alt + R:** Принудительное **Повторное сканирование с AI** с использованием вашего активного провайдера.
- **Alt + G:** Генерация и сохранение высококачественного аудиофайла (WAV/MP3). *Скрыто, если провайдер не поддерживает TTS.*
- **Alt + S / Ctrl + S:** Сохранить извлечённый текст как файл TXT или HTML.

## 3. Пользовательские промты и переменные

Вы можете управлять промтами в **Настройки > Промты > Управление промтами...**.

### Поддерживаемые переменные
- `[selection]`: Выделенный текст.
- `[clipboard]`: Содержимое буфера обмена.
- `[screen_obj]`: Скриншот объекта навигатора.
- `[screen_full]`: Полный скриншот экрана.
- `[file_ocr]`: Выбрать файл изображения или PDF для извлечения текста.
- `[file_read]`: Выбрать документ для чтения (TXT, Код, PDF).
- `[file_audio]`: Выбрать аудиофайл для анализа (MP3, WAV, OGG).

***
**Примечание:** Для всех AI-функций требуется активное интернет-соединение. Многостраничные документы обрабатываются автоматически.

## Изменения в версии 5.5 (Обновление автоматизации)

*   **AI Operator (Автономное управление - Shift+A):** Это главная жемчужина v5.5. Vision Assistant Pro перешёл от пассивного помощника к вашему личному **AI-Оператору**. Он не просто описывает экран — он берет управление на себя.
    *   *Как это работает:* Теперь вы можете отдавать голосовые инструкции для управления вашим ПК. Например, в полностью недоступном приложении, где ваш скринридер молчит, нажмите **Shift+A** и введите: *"Нажми на кнопку Настройки"* или *"Найди поле поиска, введи 'Последние новости' и нажми Enter"*. AI визуально определяет элементы, перемещает мышь и выполняет задачу за вас.
    *   *Примечание по производительности:* Эта функция оптимизирована для **Gemini 3.0 Flash (Preview)**, обеспечивая невероятно быстрые и умные ответы, способные обрабатывать даже самые сложные интерфейсы.
    *   **⚠️ Предупреждение об использовании API:** Поскольку AI-Оператору нужно «видеть», что именно происходит, для точности он отправляет скриншот высокого разрешения на каждом шаге. Учтите, что частое использование будет расходовать вашу квоту API гораздо быстрее, чем стандартные текстовые задачи.
*   **Визуальный UI Explorer (E):** Устали от «немаркированных кнопок»? Нажмите **E** для активации UI Explorer. AI просканирует всё окно и создаст список всех кликабельных элементов, которые он видит — включая иконки, графику и меню. Просто выберите пункт из списка, и AI-Оператор нажмёт на него. Это как «доступный слой» поверх любого приложения.
*   **Контекстно-зависимое умное действие с файлом (F):** Клавиша "F" была полностью переработана. Она больше не предполагает только OCR. Когда вы выбираете одно изображение, теперь он интеллектуально спрашивает о вашем намерении: вы можете выбрать **Детальное визуальное описание** для понимания сцены или **Структурированное извлечение текста (OCR)** для чтения. Меню динамически адаптируется в зависимости от типа файла и вашего активного AI-движка.
*   **Оптимизация ядра:** Мы провели глубокую очистку внутренней логики дополнения, удалив неиспользуемые устаревшие функции и избыточный код. Это делает работу более лёгкой, быстрой и надёжной для всех пользователей.
